Tổng quan bài toán

Đề bài

Nhận source code game (Cocos2d-JS client) → dùng AI Agent để:

  1. Phân tích source cũ → sinh document (design, architecture)

Mục tiêu

Phần 03

Source → Document Generation

Mục đích

Từ source code legacy (Cocos2d-JS) → sinh ra tài liệu thiết kế mà AI có thể dùng để build lại game client.

Cách làm

  1. Feed source code vào Claude → phân tích architecture, game flow, features
  2. Sinh Design docs — feature list, game flow, UI flow
  3. Sinh Architecture docs — kiến trúc hệ thống, data models
  4. Sinh Feature specs — chi tiết từng chức năng
Trick: Không đưa tất cả source code — sàng lọc nội dung trước, context lớn quá → AI không thể gen được.

Kết quả

  • AI phân tích source code legacy tương đối tốt
  • Output docs gần như đầy đủ về kiến trúc và chức năng
  • Cần human review và bổ sung chi tiết
  • Docs trở thành input chính cho các phase sau
Phân tích source code [file/folder] này. Sinh ra document bao gồm: 1. Tổng quan kiến trúc 2. Danh sách features và game flow 3. Data models 4. UI/UX flow description Format: Markdown, chi tiết đủ để developer mới hiểu và implement lại.

Phần 06

Animation & Visual Scene Understanding

🟡 Đang nghiên cứu — có kết quả ban đầu khả quan

Giải pháp: Tách agent theo chuyên môn

┌─────────────────────────────────┐ │ ORCHESTRATOR AGENT │ ├──────────┬──────────┬───────────┤ │ 📊 Logic │ 🎨 Art │ 💻 Code │ │ Agent │ Agent │ Agent │ │ │ │ │ │ Flow, │ Visual │ Implement│ │ state │ quality │ effect │ └──────────┴──────────┴───────────┘

Workflow

  1. INPUT — Capture hình ảnh + Layout art + Logic flow
  2. ANALYSIS — Art Agent phân tích → đề xuất effect, animation
  3. SCRIPTING — Ra kịch bản effect (sequence, timing, easing)
  4. IMPLEMENT — Code Agent ghép vào Godot
  5. REVIEW — Screenshot → Art Review Agent đánh giá → chỉnh sửa
  6. DONE khi đạt quality threshold
Key insight: Ràng buộc agent tốt, context ngắn, đúng trọng tâm → kết quả tốt
Phần 07

Prompt Engineering & Skill System

Approach: AI build skill cho AI

  1. Mô tả mục tiêu cho AI
  2. AI tự build skill file (SKILL.md) cho sub-agent
  3. Human review + chỉnh sửa
  4. Iterate cho đến khi đạt 10/10
Prompt chính: "Hãy xem xét, bổ sung, và tối ưu, để đạt được agent tự động 10/10"
CLAUDE.md (lean, <500 tokens) ├── Core rules: ALWAYS/NEVER ├── Project context summary └── Pointers → docs/ .claude/skills/ ├── m-ui-analyzer/SKILL.md ├── m-asset-matcher/SKILL.md ├── m-godot-scene-gen/SKILL.md └── m-docs-pdg/SKILL.md

Nguyên tắc quan trọng

1. Agent thiếu scope → hỏi lại

Không tự suy đoán — hỏi lại khi thiếu thông tin

2. Context ngắn, đúng trọng tâm

Quá nhiều context → AI confused

3. Tách logic vs art agent

Đừng bắt 1 agent làm cả 2

4. Dùng "translator agent"

Agent trung gian biên dịch ý người dùng trước khi giao task

💡 "Bản thân thường hay nói thiếu ngữ cảnh → hãy có 1 con agent ở giữa biên dịch lại để hiểu ý mình trước khi giao cho agent"

Phần 09 — 11

Bài học & Roadmap

✅ Điều nên làm (DO)

  • Luôn chạy test — đừng tin lời AI, AI xác suất
  • Ràng buộc kỹ CLAUDE.md — ALWAYS/NEVER rõ ràng
  • Tách agent theo chuyên môn — logic, art, code riêng
  • Fix ở đầu vào — metadata từ Photoshop tốt hơn detect bằng OpenCV
  • AI build skill cho AI → human review → iterate 10/10
  • Translator agent — biên dịch ý trước khi giao task
  • Invest vào tooling — MCP Docs + Editor là nền tảng

❌ Điều cần tránh (DON'T)

  • Đừng skip test — đặc biệt game logic
  • Đừng nhồi quá nhiều context — AI confused
  • Đừng 1 agent làm mọi thứ — tách ra
  • Đừng dựa MCP bên thứ 3 — custom phù hợp hơn
  • Đừng kỳ vọng AI đúng 100% — iterate là bình thường

🎯 Ngắn hạn

  • Hoàn thiện Animation/Effect pipeline
  • Document hóa MCP Godot Editor cho team
  • Team onboarding — setup workflow cho dev
  • Tự động hóa spec → implement pipeline

🚀 Dài hạn

  • Multi-agent orchestrator end-to-end
  • Visual QA agent — tự đánh giá UI quality
