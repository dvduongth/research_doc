📊 State Management

Hash-based Change Detection

{
  "concepts/ladder-mechanic.md": {
    "hash": "abc123def456",
    "processedAt": "2026-03-18T10:30:00Z"
  },
  "design/GDD-ladder-mechanic.md": {
    "hash": "xyz789ghi012",
    "processedAt": "2026-03-18T10:45:00Z"
  }
}

Logic: Nếu file hash thay đổi → re-process. Nếu giống → skip (idempotent).

🎯 Agent Coordination Pattern

1️⃣ Independent Scanning

Mỗi agent tự quét workspace theo heartbeat cycle riêng

  • agent_gd: scan concepts/ mỗi 15 phút
  • agent_dev: scan design/ mỗi 15 phút
  • agent_qc: scan design/ + src/ mỗi 15 phút

2️⃣ File-based Communication

Agents không giao tiếp trực tiếp, chỉ qua filesystem

  • agent_gd tạo GDD → file xuất hiện trong design/
  • agent_dev detect file mới → process
  • agent_qc detect cả GDD + code → test

3️⃣ State Synchronization

Mỗi agent duy trì state file riêng để track processed files

  • Hash comparison để detect changes
  • Skip unchanged files (idempotent)
  • Update state sau mỗi lần process

Hash Comparison Logic

function checkForChanges(agentId, directory, stateFile) {
  // 1. Read current state
  const state = readJSON(stateFile) || {};

  // 2. List files in directory
  const files = listFiles(directory, "*.md");

  // 3. Check each file
  const changed = [];
  for (const file of files) {
    const currentHash = md5(readFile(file));
    const storedHash = state[file]?.hash;

    if (currentHash !== storedHash) {
      changed.push(file);
      state[file] = {
        hash: currentHash,
        processedAt: new Date().toISOString(),
        agentId: agentId
      };
    }
  }

  // 4. Save updated state
  writeJSON(stateFile, state);

  return changed;
}

⚡ Error Handling & Retry Policy

Failure Detection

  • Agent Crash: OpenClaw monitors agent status, alerts sau 2 failures
  • File Access Error: Log to .state/errors.log, skip file, continue processing
  • Hash Collision: Use SHA256 thay vì MD5 để avoid collisions
  • Network Timeout: Retry 3 lần với backoff: 30s, 1m, 5m

Recovery Strategy

Error Recovery Flow:
1. Agent detects error
   ↓
2. Log error to .state/errors.log
   ↓
3. Skip problematic file
   ↓
4. Continue processing other files
   ↓
5. Send HEARTBEAT_OK to prevent false alarms
   ↓
6. Include error summary in next quality report
