Slide 5 — Gateway Routing

7-tier routing logic:
Tier 1: Peer-specific binding (user-level)
Tier 2: Parent binding (group chat hierarchy)
Tier 3: Guild + roles (Discord role-based)
Tier 4: Guild binding (server-level)
Tier 5: Team binding (workspace)
Tier 6: Account binding (user account)
Tier 7: Default agent (fallback)

Ý nghĩa:
• Có thể config nhiều agents khác nhau cho các kênh khác nhau
• Ví dụ: Support agent cho Telegram group, Moderator bot cho Discord server
• Flexibility cao — mỗi kênh/user có thể có agent khác nhau

🔄 Outer Loop

  • Check follow-up queue
  • Inject steering/pending messages
  • Call LLM (streaming)
  • Build assistant message

⚙️ Inner Loop

  • Execute tools one by one
  • Check steering messages (inject interrupt)
  • Continue until no more tools
  • Return to outer loop
Slide 6 — Dual-Loop Execution

Concept then:
Dual-loop là core của Pi-Mono agent runtime.

Outer loop: Lặp lại khi có follow-up messages (steering).
Inner loop: Thực thi các tool calls trong một turn (có thể nhiều tool).

Ví dụ flow:
1. User: "Tìm thời tiết Hà Nội"
2. Agent gọi LLM → LLM trả về tool_call weather_skill
3. Inner loop: execute weather_skill → kết quả
4. Agent gọi LLM lần nữa với kết quả → trả lời user
5. Không có follow-up → end

Steering messages:
"Pause" hoặc "Change plan" có thể inject vào giữa inner loop để dừng/skip tools đang chạy.

Key insight:
Đây là pattern để agent linh hoạt — có thể interrupt, adjust plan, và continue mà không restart.

🧠 Trust Model

Slide 8 — Security Model

8 layers:
L1 Authentication: none/token/password/trusted-proxy
L2 Role & Scopes: Hierarchical permissions
L3 Tool Policy: Whitelist tools theo agent config
L4 Approval System: Dangerous tools require human OK (critical!)
L5 Prompt Injection Defense: Wrapping + regex patterns
L6 Sandbox: Docker isolation, no sudo, non-root
L7 Secrets: No hardcoded keys, use credential store
L8 Audit: Scan for secrets, dangerous configs

Golden rule:
"Agent LUÔN là untrusted. Dù agent nói gì, dù bị injection, các lớp bảo vệ ở host/config level vẫn là tuyến phòng thủ thực sự."

Ví dụ approval workflow:
Agent muốn chạy "rm -rf /important-folder" → approval request gửi operator → operator approve/deny → chỉ execute nếu approved.

7 CVE đã fix (v2026.3.11):
WebSocket origin bypass, session hijacking, SSRF, credential leak, path traversal, webhook bypass, rate limit bypass.

Slide 10 — Data Flow Sequence

Walkthrough từng bước:
1. User gửi "Thời tiết Hà Nội?" qua Telegram
2. Channel extension nhận, convert → Gateway (WebSocket/HTTP)
3. Gateway authenticate token
4. Router resolve agent (dựa trên binding config)
5. Agent runtime nhận message, tạo session
6. Context Engine assemble context (history + current message)
7. Agent gọi LLM với tools=[weather_skill]
8. LLM quyết định gọi tool → tool_call event
9. Agent execute skill (weather_skill) — không cần approval vì read-only
10. Skill trả về data weather
11. Agent gọi LLM lần 2 với kết quả tool
12. LLM stream response text
13. Agent lưu conversation vào memory
14. Response stream ngược lại channel → user

Key points:
• Streaming: response được gửi từng chunk, không đợi full
• Tool call là async, agent quyết định execution
• Context Engine quản lý token window (sliding window)
• Memory được save sau mỗi turn

🎨 Design Patterns

OpenClaw sử dụng các mẫu thiết kế để đảm bảo tính mô-đun, mở rộng và bảo trì.

1. Gateway Hub

class GatewayHub {
  private channels = new Map<string, ChannelAdapter>();
  
  register(type: string, adapter: ChannelAdapter) {
    this.channels.set(type, adapter);
  }
  
  async send(msg: Message, to: string) {
    const adapter = this.channels.get(to.platform);
    return adapter?.deliver(msg) ?? new Error('Gateway not found');
  }
}

Mục đích: Centralize routing, unified interface.

2. Adapter

class DiscordAdapter implements ChannelAdapter {
  constructor(private client: DiscordClient) {}
  
  async deliver(msg: Message): Promise<Result> {
    return await this.client.send(this.toDiscord(msg));
  }
  private toDiscord(msg: Message): DiscordMessage {
    return { content: msg.text };
  }
}

Mục đích: Interface compatibility, legacy integration.

3. Strategy

interface RateLimitStrategy {
  shouldThrottle(userId: string): boolean;
}

class TokenBucket implements RateLimitStrategy {
  private buckets = new Map<string, TokenBucket>();
  
  shouldThrottle(userId: string): boolean {
    const bucket = this.buckets.get(userId) ?? new TokenBucket();
    return !bucket.tryConsume();
  }
}

Mục đích: Swappable algorithms, runtime selection.

4. Plugin

interface Plugin {
  name: string;
  initialize(ctx: PluginContext): void;
  dispose(): void;
}

class ModerationPlugin implements Plugin {
  name = 'Moderation';
  async initialize(ctx: PluginContext) {
    ctx.on('message', this.filter.bind(this));
  }
  private filter(msg: Message) {
    if (this.isSpam(msg)) msg.delete();
  }
}

Mục đích: Dynamic loading, isolation.

5. Approval Gate

class ApprovalGate {
  constructor(private checks: ApprovalCheck[]) {}
  
  async approve(action: Action, ctx: Context) {
    for (const check of this.checks) {
      if (!(await check.evaluate(action, ctx)).passed) {
        return { approved: false };
      }
    }
    return { approved: true };
  }
}

Mục đích: Pre-execution validation, policy enforcement.

6. Failover

class FailoverService {
  async execute(primary: () => Promise<T>, fallback: () => Promise<T>): Promise<T> {
    try {
      if (this.circuitBreaker.allow()) {
        const result = await primary();
        this.circuitBreaker.success();
        return result;
      }
    } catch (err) {
      this.circuitBreaker.fail();
      return await fallback();
    }
  }
}

Mục đích: High availability, graceful degradation.

⚠️ Anti-Patterns (Tránh!)

  • God Object — Class biến thành "tổng thể", làm tất cả
  • Spaghetti Dependencies — Circular imports, tightly coupled
  • Configuration Hell — Hard-code keys trong code
  • Empty Plugin Hooks — Interface với nhiều empty methods
  • Magic Strings — Hard-coded channel names, protocol IDs
  • Silent Failures — Không log khi plugin crash
  • Failover without Observability — Không telemetry
  • Tight Coupling to LLM — Code đội quản lý agent
Slide 12.5 — Design Patterns

Timing: 60-90 giây

Script:
"OpenClaw sử dụng 6 design patterns chính. Mỗi pattern giải quyết một vấn đề kiến trúc cụ thể."

Mỗi pattern (10-12s):
1. Gateway Hub: Centralize routing, đa nền tảng.
2. Adapter: Interface compatibility, integrate legacy systems.
3. Strategy: Swappable algorithms (rate limiting, caching).
4. Plugin: Dynamic loading, isolation, lifecycle management.
5. Approval Gate: Pre-execution validation, policy enforcement.
6. Failover: Circuit breaker, retry, graceful degradation.

Anti-patterns (20s):
"Tránh: God Object, circular dependencies, hard-coded config, empty plugin hooks, silent failures, failover without observability."

Key points:
• Patterns tạo kiến trúc mô-đun, testable, secure.
• Anti-patterns dẫn đến codebase khó maintain, buggy.
• Mỗi pattern có code example ngắn, dễ hiểu.

Q: Failover có tự động recover không?
A: Có, circuit breaker tự đóng/mở.
