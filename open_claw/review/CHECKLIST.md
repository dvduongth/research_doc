# OpenClaw Documentation Review Checklist

---

## Per-Document Quality Checks

### Structure & Completeness
- [ ] Title matches document purpose
- [ ] Introduction accessible to non-specialists
- [ ] Clear section hierarchy (H2, H3 levels)
- [ ] All sections have examples or concrete details
- [ ] Conclusion/summary present

### Technical Accuracy
- [ ] Facts align with source (OpenClaw repo, architecture)
- [ ] Version numbers consistent (2026.3.11)
- [ ] Code snippets/file paths are realistic
- [ ] Comparisons fair and accurate

### Consistency Across Docs
- [ ] Terminology consistent
- [ ] Cross-references accurate
- [ ] No contradictory statements
- [ ] Component descriptions align

### Accessibility
- [ ] Written for non-specialist audience
- [ ] Technical jargon explained on first use
- [ ] Analogies/examples provided
- [ ] Vietnamese language clear and natural

### Completeness
- [ ] Document covers stated scope
- [ ] No obvious gaps or TODOs
- [ ] All sections completed (not placeholders)
